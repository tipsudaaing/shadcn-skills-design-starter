# Dropdown & Context Menu

Command menus built on the same Radix menu primitive, styled through the
Apple-language tokens. **Dropdown Menu** (`components/ui/dropdown-menu.tsx`) opens from
a button you click; **Context Menu** (`components/ui/context-menu.tsx`) opens where you
right-click. Same items, labels, separators, checkboxes, radios, and submenus.

## Anatomy

```
┌────────────────────┐
│ Label               │  ← (Dropdown/Context)MenuLabel
│ Item          ⇧⌘P  │  ← MenuItem + MenuShortcut
│ ──────────────      │  ← MenuSeparator
│ ☑ Checkbox item     │  ← MenuCheckboxItem
│ ◉ Radio item        │  ← MenuRadioItem
│ Destructive         │  ← variant="destructive"
└────────────────────┘
```

- **Trigger** — a button (Dropdown) or a right-clickable region (Context).
- **Content** — the floating menu on `--dropdown-menu-content-bg` / `--context-menu-content-bg`.
- **Item / CheckboxItem / RadioItem / Sub** — commands, toggles, choices, nested menus.

## Usage

- **Do** use a **Dropdown** for actions tied to a visible button (a "⋯" or "Actions" trigger).
- **Do** use a **Context Menu** for power-user shortcuts on an object (a row, a canvas item).
- **Do** group with labels/separators and mark destructive items with the destructive variant.
- **Don't** hide essential actions in a context menu only — also expose them visibly.
- **Don't** hardcode colors — surfaces/items map to `--dropdown-menu-*` / `--context-menu-*` / semantic tokens.

## Token mapping

| Token | Role |
|---|---|
| `--dropdown-menu-content-bg` → `--popover` | Menu surface |
| `--dropdown-menu-item-focus-bg` → `--accent` | Highlighted item |
| `--dropdown-menu-separator` → `--border` | Divider |
| `--dropdown-menu-shortcut` → `--muted-foreground` | Shortcut hint |
| `--dropdown-menu-destructive` → `--destructive` | Destructive item |
| `--context-menu-*` | Same roles for the right-click menu |

## Accessibility (WCAG 2.2 AA)

- Radix supplies `role="menu"/"menuitem(checkbox/radio)"`, roving focus, type-ahead, and
  full keyboard control (arrows, Enter, Esc); Context Menu also opens via the keyboard menu key.
- Checkbox/radio state is exposed via `aria-checked` plus a visible mark — not color alone.
- The focused item is shown by background **and** focus state.
- Focus returns to the trigger on close.
