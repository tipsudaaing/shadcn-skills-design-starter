# Toggle & Toggle Group

Two-state buttons. **Toggle** (`components/ui/toggle.tsx`) is a single on/off button
(usually an icon); **Toggle Group** (`components/ui/toggle-group.tsx`) arranges several
into a single- or multi-select set. Both build on Radix Toggle, styled through the
Apple-language tokens, and share the pressed/hover treatment.

## Usage

- **Do** use a **Toggle** for a single formatting/state button (bold, mute, pin).
- **Do** use a **Toggle Group** `single` for mutually-exclusive options (text align).
- **Do** give icon-only toggles an `aria-label`.
- **Don't** use a Toggle for navigation or to submit вҖ” that's a `Button`.
- **Don't** hardcode colors вҖ” pressed/border map to `--toggle-*` / semantic tokens.

## Token mapping

| Token | Role |
|---|---|
| `--toggle-on-bg` вҶ’ `--muted` | Pressed / hover surface |
| `--toggle-on-fg` вҶ’ `--foreground` | Pressed icon/text |
| `--toggle-outline-border` вҶ’ `--input` | Outline variant edge |
| `--toggle-focus-ring` вҶ’ `--ring` | Focus-visible ring |
| `--toggle-radius` вҶ’ `--radius-lg` | Corner radius |

## Accessibility (WCAG 2.2 AA)

- Radix supplies `aria-pressed` (Toggle) / roving focus + `role="group"` (Toggle Group)
  and full keyboard support (Space/Enter; arrows within a group).
- Icon-only controls require an `aria-label`.
- Visible focus ring; pressed state is conveyed by fill **and** `aria-pressed`, not color alone.
- In a `single` group, exactly one item is pressed at a time.
