# Alert & Sonner

Feedback surfaces. **Alert** (`components/ui/alert.tsx`) is an inline, persistent
callout in the page flow; **Sonner** (`components/ui/sonner.tsx`) is a transient toast
that appears, then auto-dismisses. Both are styled through the Apple-language tokens.

## Anatomy

```
Alert (inline)                     Sonner (toast, transient)
┌────────────────────────────┐     ┌────────────────────────────┐
│ ✓  Heads up!        [action]│     │ ✓ Saved          [Undo]     │
│    Description text          │     │   Your changes are live.    │
└────────────────────────────┘     └────────────────────────────┘
  AlertTitle / Description / Action   toast(message, { description, action })
```

- **Alert** — `variant` `default` or `destructive`; optional icon, title, description, action.
- **Sonner** — render `<Toaster />` once near the root; call `toast(...)` to push messages.

## Usage

- **Do** use an **Alert** for persistent, in-context status the user should keep seeing.
- **Do** use **Sonner** for ephemeral confirmations ("Saved", "Copied") that shouldn't block.
- **Do** keep one clear message; add an action (Undo / Retry) when there's a next step.
- **Don't** put critical, must-act information only in a toast — it disappears.
- **Don't** hardcode colors — surfaces map to `--alert-*` / `--sonner-*` / semantic tokens.

## Token mapping

| Token | Role |
|---|---|
| `--alert-bg` → `--card` | Alert surface |
| `--alert-description` → `--muted-foreground` | Body text |
| `--alert-destructive-fg` → `--destructive` | Destructive title/icon |
| `--sonner-bg` → `--popover` | Toast surface |
| `--sonner-description` → `--muted-foreground` | Toast body |
| `--sonner-border` → `--border` | Toast edge |

## Accessibility (WCAG 2.2 AA)

- **Alert** uses `role="alert"`, so assistive tech announces it when it appears.
- **Sonner** toasts are announced via a live region; keep them brief and non-essential.
- Don't rely on color alone — pair destructive/success states with an icon and clear text.
- Toasts auto-dismiss; provide a persistent path (Alert, inline message) for anything important.
